starling.runner.stepper.Stepper¶

class starling.runner.stepper.Stepper(predictor, environment, objective, radar_sweep_period, agent, logs_directory, logs_suffix='', enable_logging=True, overwrite_logs=True, istep=0, distance_threshold=5.0, **kwds)[source]¶

Bases: object

Class that handles and combines other objects to evolve the environment, calculate the reward, log and visualise.

__init__(predictor, environment, objective, radar_sweep_period, agent, logs_directory, logs_suffix='', enable_logging=True, overwrite_logs=True, istep=0, distance_threshold=5.0, **kwds)[source]¶

Construct a Stepper instance.

Parameters

  • predictor (Predictor) – The simulator object used

  • environment (Environment) – The initial state of the environment

  • objective (Objective) – The objective used to calculate the reward

  • radar_sweep_period (float) – The sweep period for the radar, i.e. how long each radar step takes

  • agent (Agent) – The agent used to decide actions for the aircraft

  • logs_directory – The directory where logs will be written (both .csv and .json)

  • logs_suffix (String = "") – A suffix attached to all log names

  • enable_logging (Bool = True) – Whether to write .csv and .json log files

  • overwrite_logs (Bool = True) – If True, old log files in the same directory with the same name will be overwritten.

  • istep (Integer = 0) – Simulation step.

  • distance_threshold (Float = 5.0) – (temporary - will be part of new reward) Threshold distance under which separation is violated.
